# backend — OpenGOV service & data pipeline

Read order before any work here: `/PROBLEM.md` → `/docs/DESIGN.md` → `/docs/ARCHITECTURE.md` → `/docs/DATA.md`. DATA.md is the schema contract — if code disagrees with it, fix the code.

Hard rules:

- User-facing strings are Vietnamese; code, comments, and docs are English.
- Numbers (fees, deadlines, agencies, dates) render from the DB into card payloads — never from LLM output.
- Out-of-KB questions → explicit "không có trong dữ liệu" + link to the official portal. Never guess (fail-closed).
- `data/procedures/` is generated by `tools/etl/parse.ts` — fix the parser, never edit those files.
- `data/curated/`, `data/schemas/`, `data/errors/`, `data/aliases.json`, `data/golden-qa.json`, `data/provinces.json` `defunct[]` are human-reviewed — never regenerate or bulk-rewrite them.
- `backend/data/crawl/` is read-only raw input. `backend/var/` is a build artifact (gitignored).
- PII: values sent to external LLM APIs for `llm_check` are masked first.
